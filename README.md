# 리스 사이드킥 기획

버전: 0.2.6  
작성일: 2026-05-18

## 컨셉

리스 사이드킥은 RisuAI를 오래 쓰면서 생기는 작은 불편을 한 곳에서 정리하는 유틸리티 플러그인이다. 별칭은 리스킥이다. 엄청난 자동화보다는, "있으면 은근히 편한" 도구들을 모아 둔 보조 도구함에 가깝다.

초기 버전은 다음 세 기능을 제공한다.

1. 플러그인이 점점 무거워져.
2. 소악마 프롬으로 퍼메 짜면 이거 좀 지워줘.
3. 우리 이젠 헤어지자 말해요.

## 공통 원칙

- 먼저 스캔하고 결과를 보여준다.
- 사용자가 직접 대상을 고른 뒤 적용한다.
- 삭제/수정 전에는 스냅샷을 플러그인 스토리지에 저장한다.
- 실제 데이터 변경은 가능한 한 RisuAI의 기존 데이터 구조를 보존한다.
- 위험한 기능은 기본적으로 영구 삭제가 아니라 휴지통 처리 또는 선택 삭제로 제한한다.

## 기능 1. 플러그인이 점점 무거워져.

### 목표

플러그인 저장소에 쌓인 데이터를 한눈에 보고, 불필요한 항목을 선택 삭제할 수 있게 한다.

### 구현 범위

RisuAI API v3에서 접근 가능한 저장소는 크게 다음과 같다.

- `pluginCustomStorage`: 세이브 파일에 포함되는 플러그인 저장소.
- `getLocalPluginStorage()`: 기기 로컬 IndexedDB 기반 저장소.
- `safeLocalStorage`: 문자열 기반 로컬 저장소.

0.1.0에서는 DB 권한으로 확인 가능한 `pluginCustomStorage`를 우선 표시한다.

### 소유자 판정

현재 저장소 키가 항상 플러그인별 네임스페이스로 분리되는 구조는 아니므로, 소유자 표시는 확정값이 아니라 추정값이다.

- 키에 설치된 플러그인의 내부명 또는 표시명이 포함되면 해당 플러그인으로 추정한다.
- 매칭되지 않으면 `알 수 없음`으로 표시한다.
- UI에는 `소유자 추정`이라는 표현을 사용한다.

### 제공 동작

- 저장소 키 목록 표시
- 값 크기 추정
- 전체 용량 합계 표시
- 선택 항목 삭제
- 삭제 전 스냅샷 저장

### 추후 개선

- 로컬 IndexedDB 저장소까지 함께 스캔
- 저장소 키 naming convention 제안
- 플러그인별 사용량 그룹화

## 기능 2. 소악마 프롬으로 퍼메 짜면 이거 좀 지워줘.

### 목표

캐릭터의 첫 메시지와 추가 첫 메시지에서 `§...§` 형태의 의성어/연출 토큰을 제거한다.

### 대상 필드

- `character.firstMessage`
- `character.alternateGreetings[]`
- 그룹 채팅도 동일 필드가 있으면 처리 가능

### 기본 정규식

`§[^§\r\n]{1,80}§`

기본값은 한 줄 안의 짧은 토큰만 제거한다. 여러 줄에 걸친 큰 블록을 지우면 실제 서술이나 설정을 손상시킬 수 있어서 0.1.0에서는 보수적으로 처리한다.

### 제공 동작

- 현재 선택한 봇 스캔
- 전체 봇 스캔
- 제거될 조각 미리보기
- 선택 항목만 적용
- 적용 전 스냅샷 저장

## 기능 3. 우리 이젠 헤어지자 말해요.

### 목표

임포트만 해놓고 대화하지 않은 봇이나 오래 대화하지 않은 봇을 찾아 정리한다.

### 분류

- 대화 없음
- 3개월 이상 미사용
- 6개월 이상 미사용
- 1년 이상 미사용
- 날짜 불명

### 마지막 대화일 계산

다음 순서로 가장 신뢰할 수 있는 값을 사용한다.

1. 사용자 메시지 수가 0이면 무조건 `대화 없음`으로 분류한다.
2. `user` 또는 `char` 메시지의 `time` 중 최댓값을 마지막 대화일로 사용한다.
3. 메시지 시간이 전혀 없을 때만 모든 채팅의 `lastDate` 중 최댓값을 보조 기준으로 사용한다.
4. `character.lastInteraction`은 정리 판정에서 제외하고, `최근 열람/갱신 추정`으로만 별도 표시한다.

### 삭제 정책

0.1.0에서는 영구 삭제를 제공하지 않고 `trashTime = Date.now()`를 설정하는 휴지통 처리만 제공한다. 이는 RisuAI 내부의 일반 삭제 흐름과 유사하며, 실수했을 때 복구 가능성을 높인다.

### 제공 동작

- 전체 봇 스캔
- 카테고리별 필터
- 선택 항목 휴지통 보내기
- 적용 전 스냅샷 저장

## UI 구조

- 전체 화면 패널
- 상단 탭
  - 스토리지
  - 첫 메시지 청소
  - 잠든 봇 정리
- 하단 상태 메시지와 작업 결과 로그

## 0.1.0 한계

- `pluginCustomStorage`의 소유자 판정은 추정이다.
- 첫 메시지 청소는 기본적으로 한 줄짜리 짧은 `§...§`만 제거한다.
- 봇 정리는 영구 삭제가 아니라 휴지통 처리만 지원한다.
- 스냅샷은 플러그인 스토리지에 저장되며, UI에서 직접 복원 기능은 아직 제공하지 않는다.

## 0.1.1 UI 개선 기록

- 기능 탭 이름을 원래 기능명으로 교체했다.
- PowerToys처럼 좌측에 기능 카드 목록을 배치했다.
- 모바일에서는 표 대신 카드형 목록을 사용한다.
- 스토리지와 잠든 봇 정리에 컬럼 정렬을 추가했다.
- 스토리지 전체 삭제 버튼과 2단계 확인창을 추가했다.
- 권한 확인 팝업이 플러그인 창에 가려지지 않도록 권한 요청 중 플러그인 창을 잠시 숨긴다.
- 첫 메시지 청소 결과를 봇별 접기/펼치기 구조로 변경했다.

## 0.1.2 유지보수 기록

- 이후 수정마다 `//@version`, `//@display-name`, `PLUGIN_VERSION`, 스냅샷 파일명을 함께 올리는 방식으로 관리한다.
- 첫 메시지 청소와 잠든 봇 정리에 봇카드 이미지 표시를 추가했다.
- 이미지 로딩은 RisuAI `readImage()`를 우선 사용해 data URL로 변환한다.

## 0.1.3 UI 개선 기록

- 잠든 봇 정리에서도 행/카드 전체 클릭으로 체크박스를 토글한다.
- 봇카드 이미지로 행 높이가 커졌을 때 체크박스와 텍스트가 세로 가운데에 오도록 정렬했다.

## 0.1.4 UI 개선 기록

## 0.1.5 UI 개선 기록

- 첫 메시지 청소 후보 항목을 낮고 촘촘한 리스트형 카드로 정리했다.

## 0.1.6 UI 개선 기록

- 후보 항목은 중첩 테두리 대신 왼쪽 보라색 바와 배경 강조만 사용한다.

## 0.2.0 이름 변경 및 용량 개선 기록

- 플러그인 이름을 `리스 사이드킥`으로 확정했다.
- 별칭은 `리스킥`으로 사용한다.
- 잠든 봇 정리 목록에 봇 데이터와 봇카드/추가 에셋을 합산한 추정 용량을 표시한다.
- 잠든 봇 정리에서 용량 컬럼 정렬을 지원한다.

## 0.2.1 배포 경로 개선 기록

- GitHub raw 업데이트 URL을 `refs/heads/main` 경로로 변경했다.

## 0.2.2 업데이트 호환성 개선 기록

- 새 업데이트 엔드포인트를 `risu-sidekick.latest.js`로 분리했다.
- 기존 `risu-macgyver.js` 업데이트 경로를 쓰는 설치본이 새 이름으로 넘어올 수 있도록 GitHub 호환 파일을 유지한다.

## 0.2.3 잠든 봇 용량 개선 기록

- 잠든 봇 스캔 로딩창에 진행률 퍼센트를 표시한다.
- 잠든 봇 정리 안내 문구에서 휴지통 구현 세부 설명을 제거했다.
- CharX v3 익스포트 기준에 맞춰 `additionalAssets`, `emotionImages`, 메인 봇카드 이미지, 일부 음성 에셋을 용량 추정에 포함한다.

## 0.2.4 잠든 봇 스캔 속도 개선 기록

- 전체 잠든 봇 스캔에서는 에셋 파일을 실제로 모두 읽지 않고 빠른 추정값을 먼저 표시한다.
- 각 봇 행에 `정밀 계산` 버튼을 추가해, 필요한 봇만 실제 에셋 용량까지 계산할 수 있게 했다.
- 에셋 용량 캐시를 추가해 같은 에셋을 반복 계산할 때 불필요한 재읽기를 줄인다.

## 0.2.5 Small Tools 탭 추가 기록

- 네 번째 탭 `작은 도구함`을 추가했다.
- 별도 플러그인으로 분리하기 애매한 작은 UI 편의 기능을 토글로 관리하는 자리로 사용한다.
- 첫 기능으로 `사이드패널 크기 조절` 토글을 추가했다.
- 토글을 켜면 RisuAI 메인 좌측 사이드패널 오른쪽 끝에 보라색 드래그 핸들을 추가하고, 조절한 폭을 플러그인 스토리지에 저장한다.
- 설정 메뉴와 햄버거 메뉴의 버튼명은 `리스 사이드킥`으로 통일했다.
- 햄버거 메뉴에서 찾기 어려운 경우를 줄이기 위해 `action` 위치에도 `리스 사이드킥` 실행 버튼을 추가했다.
- 구형 설치본의 자동 업데이트를 위해 GitHub 배포 폴더에 `risu-macgyver.js`와 `risu-macgyver.latest.js` 호환 파일을 복구했다.

## 0.2.6 버튼 등록 안정화 기록

- 사이드패널 리사이저 초기화보다 설정 메뉴와 실행 버튼 등록을 먼저 수행하도록 순서를 바꿨다.
- Small Tools 초기화 중 오류가 나도 `리스 사이드킥` 진입 버튼은 유지되도록 초기화 실패를 격리했다.
- 햄버거 버튼 id를 기존 설치본과 호환되는 `risu-sidekick-button`으로 되돌렸다.
